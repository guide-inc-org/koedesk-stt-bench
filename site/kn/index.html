ಪೂರ್ವನೋಂದಾಯಿತ ಕ್ರಮಶಾಸ್ತ್ರದ ಅಡಿಯಲ್ಲಿ ಅಳೆಯಲಾದ ಬಹುಭಾಷಾ ಸ್ಪೀಚ್-ಟು-ಟೆಕ್ಸ್ಟ್ ನಿಖರತೆ — Track A (FLEURS), 13 ಹೋಸ್ಟ್ ಮಾಡಲಾದ ಎಂಜಿನ್ ಆವೃತ್ತಿಗಳು × 12 ಭಾಷೆಗಳು × 200 ಉಚ್ಚಾರಣೆಗಳು.

ಹಿತಾಸಕ್ತಿ ಸಂಘರ್ಷ, ಮೊದಲೇ ಸ್ಪಷ್ಟಪಡಿಸಲಾಗಿದೆ: ಈ ಬೆಂಚ್‌ಮಾರ್ಕ್ ಅನ್ನು ElevenLabs Scribe v2 ಬಳಸುವ ಡಿಕ್ಟೇಶನ್ ಉತ್ಪನ್ನ koedesk ನಿರ್ಮಿಸಿ ಹಣಕಾಸು ಒದಗಿಸಿದೆ. ನಾವು ತಟಸ್ಥರೆಂದು ನಟಿಸುವುದಿಲ್ಲ — ಬದಲಿಗೆ, ನಮ್ಮನ್ನು ಅನುಮಾನಿಸಿ ಸ್ವತಃ ಬೆಂಚ್‌ಮಾರ್ಕ್ ಅನ್ನು ಮರುಚಲಾಯಿಸಲು ಬೇಕಾದ ಎಲ್ಲವನ್ನೂ ಪ್ರಕಟಿಸಲಾಗಿದೆ: ಕ್ರಮಶಾಸ್ತ್ರ (ಮುಖ್ಯ ರನ್‌ಗಳಿಗಿಂತ ಮೊದಲೇ ಬದ್ಧಗೊಳಿಸಲಾಗಿದೆ — commit ಇತಿಹಾಸ ನೋಡಿ), ಪ್ರತಿಯೊಂದು ಕಚ್ಚಾ API ಪ್ರತಿಕ್ರಿಯೆ (results/raw), ಎಲ್ಲಾ ಸ್ಕೋರಿಂಗ್ ಕೋಡ್, ಮತ್ತು ಯಾವ ಎಂಜಿನ್‌ಗಳನ್ನು ಸೇರಿಸಬಹುದೆಂದು ನಿರ್ಧರಿಸಿದ ಸೇವಾ ನಿಯಮಗಳ ಆಡಿಟ್. Scribe ಸೋತಾಗಲೂ, ಆ ಫಲಿತಾಂಶವನ್ನು ಬದಲಾಯಿಸದೆ ಪ್ರಕಟಿಸಲಾಗುತ್ತದೆ.
Track B ಬಾಕಿ ಇದೆ. Track A ಸಾರ್ವಜನಿಕ ಪರೀಕ್ಷಾ ಸೆಟ್‌ಗಳನ್ನು ಬಳಸುತ್ತದೆ, ಇವುಗಳ ಮೇಲೆ ಎಂಜಿನ್‌ಗಳು ತರಬೇತಿ ಪಡೆದಿರಬಹುದು; ಬೆಂಚ್‌ಮಾರ್ಕ್‌ನ ಮುಖ್ಯ ಪ್ರಶ್ನೆ — ಶುದ್ಧ, ಮಾಲಿನ್ಯ-ರಹಿತ 2026ರ ಆಡಿಯೊದಲ್ಲಿ ಶ್ರೇಣಿ ಕಾಯಂ ಉಳಿಯುತ್ತದೆಯೇ? — ಎಂಬುದಕ್ಕೆ Track B ಉತ್ತರಿಸುತ್ತದೆ, ಇದನ್ನು Track A ⇄ Track B ವ್ಯತ್ಯಾಸವನ್ನು ಪ್ರಮುಖವಾಗಿ ವರದಿ ಮಾಡಿ ಇಲ್ಲಿ ಪ್ರಕಟಿಸಲಾಗುತ್ತದೆ (§9).

ಕೋಷ್ಟಕಗಳನ್ನು ಹೇಗೆ ಓದುವುದು

ಎಂಜಿನ್‌ಗಳನ್ನು ಪ್ರತಿ ಭಾಷೆಗೆ ಟೈ ಗುಂಪುಗಳಲ್ಲಿ (T1, T2, …) ಶ್ರೇಣೀಕರಿಸಲಾಗುತ್ತದೆ: ಜೋಡಿಯಾಗಿಸಿದ ಬೂಟ್‌ಸ್ಟ್ರ್ಯಾಪ್ (n=10,000, ಸೀಡ್ ಪೂರ್ವನೋಂದಣಿಯಲ್ಲಿ ಸ್ಥಿರಗೊಳಿಸಲಾಗಿದೆ) ಅದನ್ನು ಟೈರ್ ಲೀಡರ್‌ನಿಂದ ಅಂಕಿಅಂಶಾತ್ಮಕವಾಗಿ ಬೇರ್ಪಡಿಸಲಾಗದಿದ್ದಾಗ (ಜೋಡಿಯಾಗಿಸಿದ ವ್ಯತ್ಯಾಸದ 95% CI ಶೂನ್ಯವನ್ನು ಒಳಗೊಂಡಿದ್ದಾಗ) ಒಂದು ಎಂಜಿನ್ ಒಂದೇ ಟೈರ್ ಹಂಚಿಕೊಳ್ಳುತ್ತದೆ. ಒಂದೇ ಟೈರ್‌ನೊಳಗಿನ ವ್ಯತ್ಯಾಸಗಳು ಅಂಕಿಅಂಶಾತ್ಮಕವಾಗಿ ಅರ್ಥಪೂರ್ಣವಲ್ಲ — Scribe v2 ಗೂ ಸೇರಿದಂತೆ. ಜಪಾನೀಸ್, ಚೈನೀಸ್ ಮತ್ತು ಥಾಯ್‌ಗೆ ಪ್ರಧಾನ ಮೆಟ್ರಿಕ್ CER ಆಗಿದೆ, ಇತರ ಒಂಬತ್ತು ಭಾಷೆಗಳಿಗೆ WER ಆಗಿದೆ; ಎರಡನ್ನೂ ಯಾವಾಗಲೂ ತೋರಿಸಲಾಗುತ್ತದೆ. ಕಡಿಮೆ ಇರುವುದೇ ಉತ್ತಮ.

  • ಥಾಯ್‌ನಲ್ಲಿ ಪದ ವಿಭಜನೆ ಇಲ್ಲ; ಅದಕ್ಕೆ WER ವ್ಯಾಖ್ಯಾನಿಸಲಾಗಿಲ್ಲ (§5.6) ಮತ್ತು ಅದರ ಕಾಲಂ “—” ತೋರಿಸುತ್ತದೆ. CER ಥಾಯ್‌ನ ಪ್ರಧಾನ ಮೆಟ್ರಿಕ್ ಆಗಿದೆ.
  • ಖಾಲಿ ಪಠ್ಯವನ್ನು ಹಿಂತಿರುಗಿಸುವ ಎಂಜಿನ್ ಸಂಪೂರ್ಣ ಅಳಿಕೆಗಳಾಗಿ ಸ್ಕೋರ್ ಪಡೆಯುತ್ತದೆ ಮತ್ತು ಎಂದಿಗೂ ಹೊರಗಿಡಲಾಗುವುದಿಲ್ಲ (§6) — ಖಾಲಿಗಳನ್ನು ಹೊರಗಿಡುವುದು ಕೆಟ್ಟ ಎಂಜಿನ್ ಅನ್ನು ಶುದ್ಧೀಕರಿಸುವ ಶಾಸ್ತ್ರೀಯ ವಿಧಾನವಾಗಿದೆ.
  • Gemini 2.5 Pro / ಚೈನೀಸ್‌ನಲ್ಲಿ ಒಂದು ಉಚ್ಚಾರಣೆ (zh_0192) ಸೇರಿದೆ, ಇದರಲ್ಲಿ ಎಂಜಿನ್ ಒಂದು ಪ್ರತಿಲೇಖನದ ಬದಲು ಸುಮಾರು 4,800 ಅಕ್ಷರಗಳ ಇಂಗ್ಲಿಷ್ ತರ್ಕ ಪಠ್ಯವನ್ನು ಹಿಂತಿರುಗಿಸಿತು (finish_reason=stop, ಸಂಪೂರ್ಣ ಪ್ರತಿಕ್ರಿಯೆ). R-2.4/§7ರ ಪ್ರಕಾರ ಅಳೆಯಲಾಗಿ ಪ್ರಕಟಿಸಲಾಗಿದೆ: ಪೂರ್ವನೋಂದಾಯಿತ ಡೀಫಾಲ್ಟ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ಅಡಿಯಲ್ಲಿ ಎಂಜಿನ್ ವರ್ತನೆಯನ್ನೇ ಅಳೆಯಲಾಗುತ್ತಿದೆ. ಆ ಒಂದು ಉಚ್ಚಾರಣೆ ಇಲ್ಲದೆ, ಸೆಲ್‌ನ CER 0.0645 ಆಗಿರುತ್ತಿತ್ತು; ಅದರೊಂದಿಗೆ, 0.4834. ಇದು LLM-ಮಾದರಿಯ STT ಎಂಜಿನ್‌ಗಳ ನಿಜವಾದ ವೈಫಲ್ಯದ ಮಾದರಿಯಾಗಿದೆ.
  • FLEURS ಚೈನೀಸ್ ಉಲ್ಲೇಖಗಳಲ್ಲಿ 13/200 ಆಡಿಯೊದಲ್ಲಿ ಉಚ್ಚರಿಸದ ಆವರಣದೊಳಗಿನ ಇಂಗ್ಲಿಷ್ ವಿವರಣೆಗಳನ್ನು (ಉದಾ. “罗兰多·门多萨 (Rolando Mendoza)”) ಒಳಗೊಂಡಿವೆ. ಇದು ಎಲ್ಲಾ ಎಂಜಿನ್‌ಗಳಿಗೆ ಸಮಾನವಾಗಿ CER ಅನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ ಮತ್ತು ಶ್ರೇಣಿಯ ಮೇಲೆ ಪರಿಣಾಮ ಬೀರುವುದಿಲ್ಲ.
  • ಐದು Gemini ಆವೃತ್ತಿಗಳನ್ನು Cloudflare AI Gateway (ಏಕೀಕೃತ ಬಿಲ್ಲಿಂಗ್) ಮೂಲಕ ಕರೆಯಲಾಗುತ್ತದೆ; ಅವುಗಳ ಲೇಟೆನ್ಸಿ ಅಂಕಿಅಂಶಗಳು ಗೇಟ್‌ವೇ ಹಾಪ್ ಅನ್ನು ಒಳಗೊಂಡಿವೆ. ನಿಖರತೆ ಮೇಲೆ ಪರಿಣಾಮ ಬೀರುವುದಿಲ್ಲ.
  • RTF = ವಿನಂತಿಯ ವಾಲ್-ಕ್ಲಾಕ್ ಸೆಕೆಂಡುಗಳು ÷ ಆಡಿಯೊ ಸೆಕೆಂಡುಗಳು — ಕಡಿಮೆ ಇರುವುದೇ ವೇಗ. ಒಂದೇ ಯಂತ್ರದಿಂದ ಅಳೆಯಲಾಗಿದೆ (ವಿಯೆಟ್ನಾಂನಲ್ಲಿರುವ Mac mini; ಎಲ್ಲಾ ಟ್ರಾಫಿಕ್ AWS ap-northeast-1, ಟೋಕಿಯೊದಲ್ಲಿನ VPN ಎಕ್ಸಿಟ್ ನೋಡ್ ಮೂಲಕ ಹೊರಹೋಗುತ್ತದೆ — Amendment 6). ಅದೇ ಮಾರ್ಗ ಪ್ರತಿ ಎಂಜಿನ್‌ಗೂ ಅನ್ವಯಿಸುತ್ತದೆ, ಆದ್ದರಿಂದ ಸಾಪೇಕ್ಷ ಹೋಲಿಕೆಗೆ ಪರಿಣಾಮ ಬೀರುವುದಿಲ್ಲ; ಸಂಪೂರ್ಣ ಮೌಲ್ಯಗಳು ಸ್ಥಿರ VPN-ಹಾಪ್ ಓವರ್‌ಹೆಡ್ ಅನ್ನು ಒಳಗೊಂಡಿವೆ. ಮಾಹಿತಿಪೂರ್ಣ, ಶ್ರೇಣಿ ಅಕ್ಷವಲ್ಲ (§6).
  • ಬೆಲೆಗಳು ರನ್ ದಿನಾಂಕದಂದು ಪ್ರತಿ ಆಡಿಯೊ-ಗಂಟೆಗೆ ಮಾರಾಟಗಾರರ ಪಟ್ಟಿ ಬೆಲೆಗಳಾಗಿವೆ. “—” ಎಂದರೆ ಮಾರಾಟಗಾರರಿಗೆ ಗಂಟೆಗೆ ಪಟ್ಟಿ ಬೆಲೆ ಇಲ್ಲ (ಟೋಕನ್-ಆಧಾರಿತ Gemini ಆವೃತ್ತಿಗಳು; AmiVoice ಅದರ ಉಚಿತ ಹಂತದಲ್ಲಿ ಅಳೆಯಲಾಗಿದೆ).

ಪುನರಾವರ್ತಿಸಿ / ಆಡಿಟ್ ಮಾಡಿ

ಎಲ್ಲವೂ ರೆಪೊಸಿಟರಿಯಲ್ಲಿ ಇದೆ: ಎಲ್ಲಾ 28,400 ಉಚ್ಚಾರಣೆಗಳಿಗೆ ಕಚ್ಚಾ API ಪ್ರತಿಕ್ರಿಯೆಗಳು, ಸ್ಥಿರಗೊಳಿಸಲಾದ ನಾರ್ಮಲೈಜರ್‌ಗಳು ಮತ್ತು ಪರೀಕ್ಷೆಗಳೊಂದಿಗೆ ಸ್ಕೋರಿಂಗ್ ಕೋಡ್, ಕೇವಲ-ಸೇರ್ಪಡೆ ತಿದ್ದುಪಡಿ ದಾಖಲೆ, ಮತ್ತು results/scores.json-ಅನ್ನೂ ಈ ಪುಟವನ್ನೂ ನಿರ್ಣಾಯಕವಾಗಿ ಮರುಸೃಷ್ಟಿಸುವ make_scores.py / make_site.py-ಅನ್ನೂ ಒಳಗೊಂಡಿದೆ — ಕೈಯಿಂದ ಸಂಪಾದಿಸಿದ ಅಂಕಿಅಂಶಗಳಿಲ್ಲ. ಯಾವುದೇ ಸೆಲ್ ಅನ್ನು ಮರುಚಲಾಯಿಸಲು ನಿಮ್ಮ ಸ್ವಂತ API ಕೀಗಳು ಮಾತ್ರ ಬೇಕು.
